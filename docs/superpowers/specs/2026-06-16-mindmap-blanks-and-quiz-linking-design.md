# 마인드맵 빈칸채우기 + 기출문제 연결 기능 설계

날짜: 2026-06-16

## 배경

기존 사이트는 두 개의 독립된 앱으로 구성되어 있다:

- **메인 퀴즈 앱** (`index.html`, vanilla JS SPA): PDF 기출 자료실(`exams.json`, 30개), AI 채점 주관식 퀴즈(`questions.json`, 85문항), 마인드맵을 iframe으로 띄우는 화면(`s-mindmap`)으로 구성.
- **마인드맵 앱** (`mindmap-build/`, React + ReactFlow, 별도로 빌드해서 `mindmap/assets/`에 정적 산출물로 배포): `mindmapData.js` 트리에 챕터 → 서브챕터 → 리프 노드 구조로 건축시공 개념이 들어있고, 각 리프 노드는 이미 상세하게 enrich된 `explanation` 문자열을 가지고 있다(▶ 헤더, • 불릿, ★ 시험 포인트).

마인드맵 콘텐츠는 이미 충분히 enrich되어 있으므로, 이번 작업은 콘텐츠를 더 늘리는 게 아니라 **학습 방식**을 두 가지 더 제공하는 것이다:

1. 빈칸채우기로 스스로 테스트하기
2. 기출문제와 마인드맵 개념을 서로 오가며 학습하기

## 범위

### 포함

- 마인드맵 화면에 "빈칸채우기" 모드 2단계(쉬움/어려움) 추가
- `questions.json`(85문항) ↔ 마인드맵 리프 노드의 정밀 연결, 양방향 이동
- `exams.json`(PDF 30개) ↔ 마인드맵 챕터의 굵은(coarse) 연결, 단방향 이동(PDF → 마인드맵만)

### 제외 (이번 라운드에는 포함하지 않음)

- PDF 본문 OCR/텍스트 추출
- 빈칸 수동 큐레이션 (전부 자동 추출)
- 그 외 추가 학습 기능 (사용자가 "지금 2개로 충분하다"고 확인함)

## 아키텍처

```
arch-quiz-temp/                  (배포되는 정적 사이트, 메인 퀴즈 앱)
├── index.html                   (vanilla JS SPA — 모든 화면 포함)
├── questions.json                (85문항, category/keywords 필드 존재)
├── exams.json                    (PDF 메타데이터 — chapters 필드 추가 예정)
└── mindmap/
    ├── index.html                (마인드맵 React 빌드 산출물 진입점)
    └── assets/                   (빌드된 JS/CSS)

mindmap-build/                   (마인드맵 React 앱 소스, 별도 디렉토리)
└── src/
    ├── App.jsx                   (탭 전환, 빈칸 렌더러, postMessage 처리 추가)
    └── data/
        ├── mindmapData.js        (각 리프 노드에 blanks 필드 추가)
        └── linkMap.js             (신규: nodeId ↔ questionId 정적 매핑)
```

마인드맵은 메인 앱의 `<iframe>` 안에서 별도 문서로 동작하기 때문에, 두 방향의 이동은 다른 메커니즘을 쓴다:

- **퀴즈 → 마인드맵** (부모가 자식을 제어): 부모가 iframe의 `src`를 `mindmap/index.html?focus=<nodeId>` 로 바꾼다. 마인드맵 앱은 마운트 시 쿼리 파라미터를 읽어 해당 노드까지 트리를 펼치고 패널을 자동으로 연다.
- **마인드맵 → 퀴즈** (자식이 부모에게 요청): 마인드맵에서 `window.parent.postMessage({type:'openQuestion', id}, '*')` 를 보낸다. 부모는 `message` 이벤트 리스너로 받아 `qquiz` 화면으로 전환하고, 해당 문제 하나만 포함된 단일 세션을 시작한다.
- **PDF → 마인드맵 챕터**: 같은 `?focus=` 메커니즘을 챕터 id 단위로 사용 (`?focus=4` 같은 챕터 레벨 id — 리프 노드까지는 펼치지 않음).

## 기능 1 — 빈칸채우기 (쉬움 / 어려움)

### UI 구조

마인드맵 화면 상단에 모드 탭 3개를 추가한다: **📖 개념 보기 / ✏️ 빈칸(쉬움) / 🔥 빈칸(어려움)**. 트리 네비게이션과 패널은 그대로 재사용하고, 패널 안에서 `explanation`을 렌더링하는 방식만 모드에 따라 달라진다.

- 개념 보기: 기존 `renderExplanation()` 그대로
- 빈칸 모드: `explanation`의 빈칸 대상 텍스트를 `<input>`으로 치환한 인라인 렌더링 (본문 안에 입력칸이 그대로 끼어있는 형태)

### 빈칸 추출 (빌드 타임, 정규식 기반)

각 리프 노드의 `explanation`에서 추출 후보를 정규식으로 찾는다:

- 숫자+단위 패턴: `\d+(\.\d+)?\s*(mm|cm|m|MPa|%|일|시간|층|개|배|t|tf)` 등
- `★ 시험 포인트` 줄에 등장하는 핵심 용어 (이미 저자가 "이게 핵심"이라고 표시한 줄이므로 신뢰도가 가장 높음)
- 굵게 강조된 듯한 고유 기술 용어 (▶ 헤더 직후 첫 용어, 또는 시험 포인트 줄의 명사형 키워드)

난이도별 빈칸 개수:
- **쉬움**: 노드당 1~3개 (★ 시험 포인트 줄 우선)
- **어려움**: 노드당 본문 전체에서 추출 가능한 모든 숫자·핵심 용어

추출 결과는 `mindmapData.js`의 각 리프 노드에 다음 형태로 저장한다 (빌드 전 스크립트로 1회 생성, 이후 수동 보정 가능):

```js
{
  id: '5-1-1', label: 'H형강',
  explanation: '...',
  blanks: {
    easy: [{ answer: 'H형강', context: '...' }],
    hard: [{ answer: 'H형강', context: '...' }, { answer: '80MPa', context: '...' }, ...]
  }
}
```

### 채점 (느슨한 매칭)

입력값과 정답 모두 공백·조사(을/를/이/가/은/는 등 흔한 조사)를 제거하고 정규화한 후, 부분 문자열 포함 여부로 정답을 판정한다. 예: 정답 `H형강`에 대해 사용자가 `H형강이다`, `H형강(steel)`을 입력해도 정답 처리.

채점 결과는 정답이면 초록 배지, 오답이면 빨간 취소선 + 정답 병기로 인라인 표시한다.

## 기능 2 — 퀴즈 ↔ 마인드맵 연결

### 매핑 생성

`linkMap.js`는 정적 매핑 파일로, Claude가 `questions.json`의 85문항과 `mindmapData.js`의 리프 노드 내용을 직접 읽고 의미적으로 매칭해서 한 번 큐레이션한다 (런타임 키워드 알고리즘이 아님). 양방향 조회를 위해 역인덱스도 함께 만든다.

```js
// nodeId -> questionId[]
export const nodeToQuestions = { '4-4-1': ['c003', 'c017'], ... };
// questionId -> nodeId[]
export const questionToNodes = { 'c003': ['4-4-1'], ... };
```

매칭되지 않는 문제/노드가 있을 수 있으며, 이 경우 관련 UI(버튼/섹션)를 그냥 숨긴다.

### 퀴즈 → 마인드맵

`qq-result` 화면(AI 채점 결과)에 "🗺️ 관련 개념 보기" 버튼을 추가한다. 클릭 시 `go('mindmap')`으로 전환하면서 iframe의 `src`에 `?focus=<nodeId>`를 붙인다.

### 마인드맵 → 퀴즈

마인드맵 노드 패널 하단에 "🔗 관련 기출문제" 섹션을 추가한다. `linkMap.js`에서 매칭된 문제 1~2개를 카드(카테고리 배지 + 문제 앞부분 미리보기)로 보여주고, 클릭 시 `postMessage`로 부모에게 알려 `qquiz` 화면으로 전환, 해당 문제 하나만 포함된 단일 세션을 시작한다 (AI 채점 흐름은 기존 그대로 작동).

## 기능 3 — PDF 기출 ↔ 마인드맵 챕터 (굵은 연결)

Claude가 30개 PDF의 라벨(연도/회차)을 보고 일반적인 건축기사 실기 출제 경향에 따라 각 시험에 마인드맵 최상위 챕터 id를 태깅한다 (`exams.json`에 `chapters: ["4", "7"]` 필드 추가). PDF 본문을 직접 읽고 추출하는 것이 아니므로 정확도는 완벽하지 않을 수 있고, 추후 실제로 풀어보면서 보정 가능하다는 점을 사용자가 인지하고 동의함.

PDF 시험지 화면(`s-viewer`)의 "📄 기출 자료" 카드 아래에 챕터 칩을 나열한다 (`4.철근콘크리트공사`, `7.방수공사` 등). 클릭 시 마인드맵 화면으로 전환되고 해당 챕터까지만 펼쳐진 상태로 보여준다 (리프 노드 자동 오픈은 하지 않음 — 챕터 단위 연결이기 때문).

## 빌드/배포 파이프라인

기존 파이프라인과 동일한 방식 유지:

1. `mindmap-build/src/data/mindmapData.js`에 `blanks` 필드, `linkMap.js` 신규 생성
2. `App.jsx`에 모드 탭, 빈칸 렌더러, `?focus=` 파라미터 처리, `postMessage` 송수신 추가
3. `npm run build` → 새 해시의 JS를 `arch-quiz-temp/mindmap/assets/`로 복사 → `mindmap/index.html`의 script src 갱신
4. 메인 `index.html`에 qquiz 결과 버튼, `message` 이벤트 리스너, qquiz 단일문제 모드, PDF 뷰어 칩 추가
5. `questions.json`은 변경 없음, `exams.json`에 `chapters` 필드만 추가
6. 커밋 → 사용자 확인 후 푸시

## 검증 방법

- 마인드맵 단독: `npm run dev`로 구동 후 — 탭 전환, 빈칸 채점 정/오답 표시, `?focus=`로 진입했을 때 자동 펼침/패널 오픈 확인
- 메인 앱: 정적 HTML이므로 브라우저에서 직접 열어 iframe ↔ 부모 간 postMessage 통신, 버튼 동작, 단일문제 세션 동작 확인
- 두 앱 다 별도 자동 테스트 스위트가 없으므로(정적 사이트), 수동 브라우저 확인이 검증의 전부
